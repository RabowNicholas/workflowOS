# REFERENCE

Global locked rules for WorkflowOS. Voice, banned language, quality tests, and how to work with Nick. Pull for any task involving voice, copy, content, brand, or output quality.

Project-specific rules live in each project folder's REFERENCE.md. Course structure decisions are in course/REFERENCE.md.

NOTE: Voice is seeded from the WorkflowOS Starter Vault v1 and v2 PDFs, which are currently the closest thing to canonical WorkflowOS writing. It will be refined as the first courses get built. Treat voice as in-flux until locked.

<voice>
Direct. Builder-to-builder. Second person. Short declarative sentences, often fragments. No hedging, no thought-leader gloss, no corporate jargon.

Three-beat rhythms are common: "Build the three files. Run the protocol. See what changes."

Contrast structures pull weight: state the thing, then state what it is not. "The vault is the source of truth. Not your memory, not chat history, not whatever was said last time."

Concrete over abstract. If the point can be made with a real object or a real action, use it. "Running a powerful engine with no memory." "Treat Claude like a search engine."

Confessional origin beats pitch-deck framing. "I built this for myself first" works. "Introducing a revolutionary new approach" does not.

Colons do emphasis work: "If you take one thing from this PDF, take this:"

GOOD EXAMPLES (pulled from Starter Vault v2):
- "You keep starting over."
- "A vault, not a chat."
- "The problem isn't Claude."
- "Claude works dramatically better when it has a vault to read from, not a chat to guess from."
- "Build the three files. Run the protocol. See what changes."

BAD EXAMPLES (what WorkflowOS should never sound like):
- "Introducing the revolutionary new way to leverage AI."
- "Unlock the full potential of your workflow with our seamless, best-in-class solution."
- "In today's fast-paced world, AI is a game changer."
</voice>

<banned>
Never use: unlock, leverage, seamless, revolutionary, game changer, best-in-class, "in today's fast-paced world." These read as SaaS marketing or generic thought-leader copy and are the exact cadence the WorkflowOS voice rejects.

Also avoid the construction "it's not just X, it's Y" — it's a ChatGPT tell and weakens any claim that follows it.
</banned>
